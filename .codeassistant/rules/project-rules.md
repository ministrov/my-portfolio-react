## Общее

| Что               | Кратко                                                   |
| ----------------- | -------------------------------------------------------- |
| **Фреймворк**     | Create React App (CRA)                                   |

| **Стили**         | Обычные CSS с БЭМ-неймингом                              |

| **Комментарии**   | комментарии и описания — **на русском**                  |

## 1. Архитектура проекта

1. Исходники находятся в `src/`. Основные папки:
   - `src/` – корневая папка исходного кода
   - `components/` – переиспользуемые компоненты
   - `layouts/` – компоненты макета (Header, Footer)
   - `pages/` – страницы приложения
   - `sections/` – крупные секции страниц
   - `hooks/` – кастомные хуки
   - `context/` – контексты React (например, LanguageProvider)
   - `utils/` – вспомогательные функции (i18n, утилиты)
   - `const/` – константы
   - `assets/` – статические ресурсы (изображения, шрифты, PDF)
   - `styles/` – глобальные стили
   - `public/` – статические ресурсы для публикации

2. Каждый **компонент** – в отдельной папке `ComponentName/` внутри `components/` (например, `components/errorMessage/`), содержащей:
   - `ComponentName.jsx` – основной компонент
   - `style.css` – стили компонента
   - (опционально) `ComponentName.test.jsx` – тесты
   - использовать дефолтный экспорт по примеру:  `const ErrorMessage = ({ ... }) => { ... }
   
   export default ErrorMessage;
   `

## 2. React

* **Только** функциональные компоненты, классовые компоненты запрещены.
* PascalCase для имён файлов и компонентов (`ErrorMessage.jsx`).
* Props неизменяемы; стейт всегда обновляем через set‑функции.
* Внешние эффекты – в `useEffect` с корректным списком зависимостей.
* Кастомные хуки – в папке `hooks/`, имя начинается с `use` (`useLanguage.jsx`).
* Для условного рендеринга используйте тернарный оператор или `&&`.

## 3. JavaScript (JSX)

* Все файлы компонентов и утилит должны иметь расширение `.js` или `.jsx`.
* Для пропсов компонентов можно использовать PropTypes или опираться на структуру.
* Избегать `any`. Использовать конкретные типы через JSDoc при необходимости.
* Импорты должны быть упорядочены: сторонние библиотеки, затем внутренние модули.

## 4. Стили (CSS с БЭМ)

* Стили компонента – в файле `style.css` в той же папке.
* Имена классов – в стиле БЭМ (`block__element--modifier`). Примеры:
  - ✅ `.not-found`, `.not-found__image`, `.not-found__btn`
  - ❌ `.card`, `.image`, `.price` (если они не следуют БЭМ)
* Глобальные стили – в `src/styles/main.css`.
* **Не** использовать инлайн-стили (`style={{ ... }}`), кроме динамических значений (например, `width: ${width}px`).
* **Не** использовать препроцессоры (Sass/Less) без согласования.

## 5. Состояние (Context API)

* Глобальное состояние – через React Context (например, `LanguageProvider`).
* Контексты хранить в `context/`.
* Для сложного состояния можно использовать комбинацию `useReducer` и Context.
* Не дублировать состояние: если данные уже в контексте, не хранить локально.

## 6. Маршрутизация (React Router)

* Страницы – в `src/pages/` (например, `HomePage.jsx`).
* Layout – `src/layouts/Layout.jsx`.
* Динамические маршруты – через `react-router-dom` (например, `:id`).
* Загрузка данных – через `useEffect` в Client Components.
* Обработка ошибок – через `ErrorBoundary` или отдельные страницы (например, `PageNotFound`).

## 7. Анимации (Framer Motion)

* Для анимаций использовать `framer-motion`.
* Компоненты `motion.div`, `motion.span` и т.д.
* Плавные переходы между состояниями – через `animate` и `transition`.

## 8. Именование и структура

* Папки и файлы – PascalCase (например, `ErrorMessage`).
* Компоненты – PascalCase (`ErrorMessage`).
* Хуки – camelCase с префиксом `use` (`useLanguage`).
* Функции и переменные – camelCase.
* Константы – UPPER_SNAKE_CASE.
* CSS-классы – БЭМ (`.block__element--modifier`).

## 9. Код-стайл и линтинг

* Используется ESLint с конфигурацией Create React App.
* Перед коммитом запускать `npm run lint`.
* Форматирование – Prettier (опционально, есть конфиг `.prettierrc`).
* Максимальная длина строки – 100 символов.
* Использовать одинарные кавычки для строк.
* Завершающая запятая в объектах и массивах.

## Ни в коем случае не добавляй:

- Class компоненты
- Styled Components / Emotion (только обычные CSS)
- Другие стейт-менеджеры (Redux, MobX, Zustand) без согласования
- Tailwind CSS
- Инлайн-стили на элементах (кроме динамических)
- Глобальные CSS-классы для стилизации компонентов (кроме утилитарных)
- UI-библиотеки (MUI, Ant Design) без согласования
- Дополнительные зависимости без обсуждения
- TypeScript без миграции проекта
