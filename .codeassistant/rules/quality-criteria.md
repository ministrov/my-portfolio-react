## Критерии качества разработки на React

### БАЗОВЫЕ КРИТЕРИИ (ОБЯЗАТЕЛЬНЫЕ)

| Критерий | Описание |
|----------|----------|
| **Б1. Соответствие ТЗ** | Все обязательные пункты технического задания выполнены |
| **Б2. Необработанные ошибки** | При работе сайта не возникает необработанных ошибок, программа не ломается и не зависает |
| **Б3. Кроссбраузерность** | Код работает без ошибок в Chrome, Firefox, Safari, Edge (последние 2 версии) |
| **Б4. ESLint** | Код проходит проверку ESLint без ошибок |
| **Б5. camelCase** | Переменные, параметры, методы начинаются со строчной буквы в camelCase |
| **Б6. Английские существительные** | Названия переменных — английские существительные, без сокращений (кроме xhr, evt, i/j/k, cb) |
| **Б7. UPPER_SNAKE_CASE для констант** | Константы — прописные буквы с подчёркиваниями (MAX_HEIGHT) |
| **Б9. Enum с заглавной буквы** | Перечисления — существительные с заглавной буквы (CamelCase), значения — CamelCase |
| **Б10. Массивы — мн. число** | Массивы названы существительными во множественном числе или с венгерской нотацией |
| **Б11. Глагол в названии функции** | Название функции/метода содержит глагол (get, print, fire) |
| **Б12. React Router** | Для работы с адресной строкой используется только React Router |
| **Б13. Redux** | Для хранения глобального состояния используется только Redux |
| **Б16. Нет мёртвого кода** | Нет неиспользуемых файлов, модулей, закомментированных участков кода |
| **Б18. Директория src** | Исходный код — в src/, конфиги — в корне |

### ДОПОЛНИТЕЛЬНЫЕ КРИТЕРИИ (ДЛЯ ЗАЩИТЫ НА 100%)

#### ⚙️ РАБОТОСПОСОБНОСТЬ

| Критерий | Описание |
|----------|----------|
| **Д1. Полное ТЗ** | Все обязательные и необязательные пункты технического задания выполнены |

#### 📝 СТИЛЬ КОДА

| Критерий | Описание |
|----------|----------|
| **Д2. handle-обработчики** | Функции-обработчики событий названы через handle + объект + событие (handleFormSubmit) |
| **Д3. Константы в Enum** | Составные константы собраны в перечисления (Enum) или константный объект |
| **Д5. Нет магических значений** | Под каждое значение заведена переменная-константа, под тип — псевдоним типа |

#### ⚛️ REACT

| Критерий | Описание |
|----------|----------|
| **Д8. Структура TSX-файла** | Импорты → описание props → код компонента |
| **Д9. on-колбэки** | Колбэки, переданные в props, названы через on (onClick, onSubmit) |
| **Д11. Ресурсы компонента** | Стили и другие ресурсы компонента размещены в его директории |

#### 🗃️ REDUX

| Критерий | Описание |
|----------|----------|
| **Д12. Action Creators** | Для создания Actions используется паттерн Action Creators |
| **Д13. redux-thunk** | Для работы с асинхронными actions используется библиотека redux-thunk |
| **Д14. Чистый редьюсер** | Редьюсер не содержит побочных эффектов, опирается только на state, action и чистые функции |
| **Д15. Редьюсер — существительное** | Для именования функции-редьюсера используются только существительные |
| **Д16. Логика в редьюсере** | Логика изменения состояния описывается в редьюсере, а не в компоненте |
| **Д17. Разбивка редьюсеров** | Нет «универсального редьюсера», редьюсеры разбиты и объединены через combineReducer |
| **Д18. Селекторы** | Для чтения состояния применяются селекторы, при необходимости — мемоизация (reselect) |
| **Д19. Типы действий** | Паттерн для action type: домен/действие (user/login) |

#### 🪝 HOOKS

| Критерий | Описание |
|----------|----------|
| **Д20. isMounted в useEffect** | При изменении состояния в useEffect выполнена проверка на демонтирование компонента |
| **Д21. Префикс use** | Кастомные хуки начинаются с префикса use (useUser) |

#### 🧪 ТЕСТЫ И КОРРЕКТНОСТЬ

| Критерий | Описание |
|----------|----------|
| **Д22. Тесты редьюсеров** | Существуют юнит-тесты на все состояния всех редьюсеров |
| **Д23. Тесты компонентов** | Все компоненты в проекте покрыты тестами |
| **Д24. Расположение тестов** | Тестовые файлы лежат рядом с тестируемыми файлами, названы *.test.js |
| **Д27. Моковые данные** | В тестах используются моковые данные, только важные для тестирования props |
| **Д28. Нет прямого обращения к DOM** | Нет document.querySelector, используется ref |
| **Д29. Нет вычисляемых полей в стейте** | Глобальный стейт не содержит значений, вычисляемых из других полей |
