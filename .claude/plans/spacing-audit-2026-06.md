# Аудит отступов и пропорций — июнь 2026

Анализ по четырём скриншотам: `390×844`, `768×1024`, `1366×768`, `1920×1080`.
Все значения в rem (1rem = 16px) и px.

---

## Критические проблемы

### C1 — wrapper margin-top на мобильном: 16px при высоте хедера ~64px

**Файл:** `src/styles/main.css`

`.wrapper { margin-top: 16px }` при `max-width: 768px` — в то время как
`.header` на мобильном имеет `padding: 1rem 1rem` (16+16px) + контент (~32px) = ~64px высоты.
Когда хедер становится fixed (`.header--bg`), контент уезжает под него на 48px.
Hero спасает собственный `padding-top: 40px`, но другие страницы (About, Projects) уязвимы.

**Задача:**

- [ ] `styles/main.css` — изменить мобильный `margin-top: 16px` → `margin-top: 64px`

---

### C2 — Резкий скачок контейнера на 1268px: 1440px → 1068px (–372px)

**Файл:** `src/styles/main.css`

На ноутбуке 1366px контейнер = 1068px, что оставляет (1366–1068)/2 = **149px пустых боковых полей**.
На десктопе 1920px поля = (1920–1440)/2 = **240px**.
Разрыв между ноутбуком и десктопом выглядит как резкий сдвиг контента.

**Задача:**

- [ ] `styles/main.css` — добавить промежуточный брейкпоинт `max-width: 1520px`
      чтобы сгладить переход: `1440px → 1200px → 1068px` вместо прямого прыжка

визуального разделения: все три на белом фоне без border-radius, без фона.
Остальные секции чередуются бело/серо, но здесь три белых подряд.

**Задача:**

- [ ] Добавить `background-color: var(--color-gray-600)` и `border-radius: 42px`
      к секции `.services` — она встаёт между двумя белыми секциями и создаст ритм:
      white (projects) → gray (services) → white (testimonials)
- [ ] На `max-width: 768px` — `border-radius: 16px` как у остальных

## Hero-секция

### H1 — Hero min-height 78vh создаёт избыточный вертикальный воздух

На 1080px viewport: 78% = 842px высоты hero.
Контент (kicker + заголовок + подзаголовок + кнопка) ≈ 350px.
Остаётся ~250px воздуха сверху и снизу внутри секции.
На 768px и ниже `min-height: auto` — там всё OK.

**Задача:**

- [ ] `sections/hero/style.css` — уменьшить `min-height: 78vh` → `min-height: 70vh`
- [ ] Это снижает пустой воздух примерно на 65px, не ломая визуальный ритм

## Заголовки секций

### S1 — Heading font-size 3.8rem на 390px занимает >80% ширины

**Файл:** `src/components/heading/style.css`

`.heading-sec__main { font-size: 3.8rem }` = 60.8px при ширине контента 358px (390 – 2×16px).
Одно слово «ПРОЕКТЫ» (7 букв Oswald) занимает почти всю строку.
На ≤575px размер уже снижается до 3rem (48px) — но 575px это слишком широкое пороговое значение.

**Задача:**

- [ ] `components/heading/style.css` — добавить брейкпоинт `max-width: 480px`:
      `font-size: 2.75rem` (44px) — более пропорционально на 390px

---

### S2 — heading-sec\_\_mb-med (4rem) применяется непоследовательно

`margin-bottom: 4rem` (64px) задаётся классом `.heading-sec__mb-med`.
Некоторые секции используют его, другие — базовые `2rem` (32px).
На скриншотах это выглядит как разный воздух между заголовком и контентом
в разных секциях — заметно на ноутбуке и десктопе.

**Задача:**

- [ ] Провести аудит: какие секции используют `mb-med`, какие нет
- [ ] Унифицировать: либо все получают `4rem`, либо все `3rem` с новым классом

## About-секция

### A1 — About на планшете 768px: горизонтальный ряд фото+текст выглядит тесно

**Файл:** `src/sections/about/style.css`

На `≤1104px` `.about__left` переходит в `flex-direction: row` с `gap: 1.5rem`.
На 768px фото (~120px) + gap (24px) + имя+соцсети укладываются в ~260px,
оставляя правой части ~484px. Gap между колонками при `gap: 3rem` (48px) может
быть тесным.

**Задача:**

- [ ] `sections/about/style.css` — добавить для `max-width: 900px` или через
      корректировку существующего `max-width: 1104px` блока:
      уменьшить `gap: 3rem → gap: 2rem` для `about__wrapper` на 768–1104px

## Внутренние отступы секций

### I1 — section padding 6rem/4rem консистентен, но heading gap разный

Все секции: `padding-top/bottom: 6rem` (96px) desktop, `4rem` (64px) mobile.
Это ОК и консистентно. Проблема в пространстве между заголовком и контентом:

| Секция       | Отступ заголовка → контент             |
| ------------ | -------------------------------------- |
| Projects     | 4rem (mb-med)                          |
| Services     | 4rem (mb-med)                          |
| Advantages   | 4rem (mb-med)                          |
| Testimonials | не задан явно                          |
| FAQ          | не задан явно                          |
| Contact      | margin-top: 3rem на .contact\_\_layout |

**Задача:**

- [ ] Проверить и выровнять gap «заголовок → контент»: все секции должны использовать
      одинаковый принцип (рекомендую 3rem для всех вместо смеси 4rem/3rem/auto)

---

### I2 — FAQ accordion items: max padding 2.5rem = 80px на строку на десктопе

**Файл:** `src/components/accordionItem/style.css`

`padding-block: clamp(1.25rem, 0.9rem + 1.8vw, 2.5rem)` достигает 2.5rem (40px)
на широких экранах. При 6+ вопросах FAQ-секция занимает ~560px только на padding.
На 1920px это выглядит растянуто.

**Задача:**

- [ ] Уменьшить max значение clamp с `2.5rem` → `1.75rem`:
      `clamp(1.25rem, 0.9rem + 1.8vw, 1.75rem)`

---

### I3 — Services: container padding-inline 16px на 390px слишком узкий для карточек

**Файл:** `src/styles/main.css`

`clamp(16px, 4vw, 32px)` даёт 16px padding-inline на 390px.
Контентная ширина: 390 – 32 = **358px**.
Сервисные карточки с собственным `padding: 24px` внутри получают
358 – 48 = **310px для текста** — довольно тесно для описания услуги.

**Задача:**

- [ ] Не менять `clamp` глобально, но для `.services__description` на мобильном
      ограничить `font-size: 15px` вместо 18px при `max-width: 475px`
