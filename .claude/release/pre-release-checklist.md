# Pre-Release Checklist for my-portfolio-react

Адаптированный чеклист для одного разработчика + Claude перед релизом портфеля.

## ✅ Функциональность & Тестирование

- [ ] **Ручное тестирование всех страниц** — HomePage, AboutPage, ProjectsPage на локальном `npm start`
  - Навигация между всеми маршрутами работает
  - Нет белых экранов, ошибок в консоли браузера
- [ ] **Проверка на мобильных размерах** — открыть DevTools, протестировать на SM/MD/LG/XL (в Chrome достаточно)
  - Все текст читаемо
  - Кнопки, формы кликабельны
  - Меню мобильное корректно открывается/закрывается
- [ ] **Проверка всех интерактивных элементов:**
  - Язык переключается корректно (RU ↔ EN)
  - Фильтр проектов работает (все комбинации skills)
  - Форма контакта отправляется (если есть backend) или валидирует input
  - Карусель проектов свайпится (Swiper)
  - Ссылки на соцсети / внешние ресурсы открываются
- [ ] **Проверка в разных браузерах** — минимум Chrome, Firefox
  - Если есть доступ: Safari (через BrowserStack free tier или знакомого с Mac)
  - Для портфеля достаточно этих трёх

## ✅ Перформанс & Оптимизация

- [ ] **Запустить production build**: `npm run build`
  - Build успешен без ошибок/warnings
  - Размер бандла приемлем (проверить терминал)
- [ ] **Анализ бандла**: `npm run analyze`
  - Нет неожиданно больших зависимостей
  - Lazy-loaded chunks (AboutPage, ProjectsPage) действительно отделены
  - Source map сгенерирован
- [ ] **Lighthouse скор**: в Chrome DevTools → Lighthouse
  - Performance ≥ 80
  - Accessibility ≥ 85
  - Best Practices ≥ 90
  - SEO ≥ 95 (критично для портфеля)

## ✅ Code Quality

- [ ] **Проверить linting**: `npx eslint src`
  - Нет ошибок (errors)
  - Все warnings исправлены или осознанно проигнорированы
- [ ] **Консоль браузера чистая** при запуске (`npm start`):
  - Нет console.error, console.warn (кроме intentional)
  - Нет React warnings (StrictMode скажет о проблемах)
  - Нет undefined errors при навигации

## ✅ i18n & Локализация (Критично для вас!)

- [ ] **Все текст в переводах, не захардкодирован**
  - Проверить что текст берётся из `src/utils/i18n/locales/en/en.json` и `ru/ru.json`
  - Нет missing translation keys (должны быть видны в app или в консоли)
- [ ] **Обе локали полностью протестированы**:
  - Переключить язык на RU — все видимо на русском
  - Переключить язык на EN — все видимо на английском
  - Переключение работает из любого места (Header, URL param)
- [ ] **Language persistence** — перезагрузить страницу, язык сохранился
- [ ] **SEO для обеих локалей**:
  - Meta title/description в обе локали добавлены (react-helmet-async)
  - Open Graph теги на обе локали (если шеришь портфель в соцсетях)

## ✅ Безопасность & Правильность

- [ ] **Нет exposed secrets**
  - Нет API keys, паролей, приватных токенов в коде
  - Если используется backend — проверить что запросы идут на HTTPS
- [ ] **Dependency security**: `npm audit`
  - Нет critical уязвимостей
  - Если есть warnings — известны ли они, приемлемы ли
- [ ] **Form validation** (контактная форма):
  - Обязательные поля валидируют перед отправкой
  - Email формат проверяется
  - Нет XSS в пользовательском input (он экранирован React, но проверь)

## ✅ Данные & Содержание

- [ ] **Все контактные данные актуальны**:
  - Email правильный
  - Социальные ссылки работают и актуальны
  - Резюме / CV свежее (если оно есть)
- [ ] **Проекты в портфеле актуальны**:
  - Описания верны
  - Ссылки на репо/демо работают
  - Скриншоты / превью кажутся хорошо
  - Skills теги верны

## ✅ Перед финальным деплоем

- [ ] **Последний раз посмотреть git diff** перед пушем
  - Нет случайно коммитнутого дебага кода
  - Нет изменений `.env` или приватных файлов
- [ ] **Коммит финализации** (если нужен):
  - Сообщение типа `release: v1.0.0` или `chore(release): prepare for launch`
  - Все изменения стейджированы
- [ ] **Финальный `npm start`** — всё работает как ожидается
- [ ] **Production build** финально проверен локально (`npm run prod`)

---

## Порядок выполнения

1. **Функциональность & тестирование** (самое важное)
2. **Code Quality** (linting, консоль)
3. **i18n** (для вас критично)
4. **Перформанс** (Lighthouse, bundle size)
5. **Безопасность & данные** (финальная проверка)
6. **Деплой** (когда всё пройдено)

---

**Когда что-то не в порядке:**

- Linting ошибка? → `npx eslint src --fix`
- Performance плохо? → Claude поможет оптимизировать
- i18n ключ пропущен? → добавить в оба файла локалей (en.json + ru.json)
- Браузер ошибка? → DevTools → исправить
